# IRTC 1.1.0 开发计划(GPCM 易用性与 Q 矩阵版)

日期:2026-07-17　负责人:Ma Kunxiang　目标版本:1.1.0　开发分支:v1.1-dev

## 一、目标与原则

在 1.0.0 已验证的易用层之上,围绕 GPCM(及多维模型)的实际使用流程补齐五块能力:
样本权重同步读取、Q 矩阵多格式读取与题目对齐校验、GPCM 部分计分的端到端支持
(Q 矩阵声明 → 答案文件计分 → 参数输出语义标注)、极端作答数据的稳健处理、
更专业更详细的报告。

沿用 1.0 全部原则:核心估计层(`irtc.mml`/`irtc.mml.2pl`)零改动、零新增依赖;
新依赖仅进 Suggests;原 API 完全向后兼容;新函数 `irtc_*` 命名;双语输出;
R 源码 ASCII-only(\uXXXX 转义)。

已确认的设计决策(2026-07-17 与负责人确认):

1. Q 矩阵与作答数据题目不一致:默认「警告 + 自动对齐(取交集)」,
   提供 `on_mismatch="error"` 严格模式。
2. 无人得到的分数类别:默认自动折叠类别并在输出中标注;
   另提供贝叶斯先验方案保留原类别结构(`rare_categories="prior"`)。
3. 报告增强:模型诊断章节 + 数据处理透明度章节(不加图表嵌入与方法学章节)。
4. 权重:自动识别常见列名 + `weights=` 参数显式指定。

## 二、功能模块与执行细节

### M1 样本权重读取(irtc_read + irtc)

目标:样本表格中的权重列与作答数据同步读入,并自动进入估计。

- `irtc_read()` 新增参数 `weights=NULL`(列名或列号)与自动识别。
  - 自动识别列名池(不区分大小写、去空白):
    `weight, weights, wt, wgt, w, pweight, sampling_weight, 权重, 样本权重,
    加权, 权数, 抽样权重`。命中后从作答数据中分离,存入返回对象
    `$weights`,记日志(新码 I115「识别到权重列」/ I116「使用指定权重列」)。
  - 校验:必须可转数值,否则报错(新码 E107);NA 权重 → 警告(W117)并置 1;
    ≤0 的权重 → 报错(E108,权重必须为正)。
  - `print.irtc_data` 增加一行显示是否携带权重及加权样本量
    (sum(weights))。
- `irtc()` 将 `data_obj$weights` 作为 `pweights` 传入
  `irtc.mml`/`irtc.mml.2pl`;若用户在 `...` 中显式提供 `pweights`
  则以用户为准并警告二者同时存在(W418)。
- 剔除空行时权重与 pid 同步对齐(复用 `rows_kept` 机制)。

涉及文件:`R/irtc_read.R`、`R/irtc.R`、`man/irtc_read.Rd`、`man/irtc.Rd`、
新测试 `tests/testthat/test-read-weights.R`。

### M2 Q 矩阵读取与对齐(新文件 R/irtc_qmatrix.R)

目标:从多种类型文件直接读取 Q 维度矩阵,校验其题目与样本题目是否有出入,
并支持声明某题是否分部计分。

- 新函数 `irtc_read_q(x, sheet=1)`:
  - 输入:文件路径(复用 `irtc_read_file`,支持
    .xlsx/.xls/.csv/.tsv/.txt/.dat/.sav/.por/.dta/.sas7bdat/.xpt)
    或 data.frame/matrix。
  - 表格格式约定(写入文档与手册):
    - 题目 ID 列:名为 `item/item_id/题目/题号/题目编号` 之一,
      或默认取第一个非数值列;
    - 维度列:其余数值(0/1 或载荷)列,列名即维度名
      (如 `代数`、`几何`),将用于后续所有输出表头;
    - 可选分部计分列:名为 `partial/分部计分/多级计分/max_score/满分/最高分`
      之一。取值为 0/1、TRUE/FALSE、是/否(是否分部计分),
      或整数满分(≥2 即视为分部计分,同时记录满分)。
  - 校验:维度列至少 1 列且数值;每题至少载荷一个维度(否则 E5xx);
    题目 ID 无重复;分部计分列取值合法。
  - 返回 S3 对象 `irtc_qmatrix`:`$Q`(items×dims 数值矩阵,带 dimnames)、
    `$partial`(命名逻辑向量)、`$max_score`(命名整数向量,NA 表示未声明)、
    `$log`;提供 `print` 方法。
- 新函数 `irtc_align_q(data_obj, q, on_mismatch=c("warn","error"))`:
  - Q 有而样本无的题:警告(新码 W420)逐题列出,从 Q 中移除;
  - 样本有而 Q 无的题:警告(W421)逐题列出,从作答数据中移除;
  - `on_mismatch="error"` 时任一不一致即报错(E422),错误 data 中附完整差异;
  - 对齐后 Q 行序重排为与作答数据列序一致;全部动作写入清洗日志。
- `irtc()` 新增参数 `q=NULL, on_mismatch="warn"`:接受路径 / data.frame /
  `irtc_qmatrix`;内部读取 → 对齐 → `Q` 传入估计函数;`q` 与 `...` 中显式
  `Q` 同时出现时以用户 `Q` 为准并警告。

涉及文件:新 `R/irtc_qmatrix.R`、`R/irtc.R`、`man/irtc_read_q.Rd`、
`man/irtc_align_q.Rd`、新测试 `tests/testthat/test-qmatrix.R`。

### M3 答案文件读取与 GPCM 部分计分计分(irtc_score 扩展)

目标:判分所需的正确答案支持从多类型文件读取,并支持"部分正确"答案。

- `irtc_score()` / `irtc()` 的 `key` 与 `rules` 参数均接受文件路径
  (复用 `irtc_read_file`)。
  - key 文件格式:列 `item/题目/题号` + `answer/key/答案/正确答案`;
    可选列 `partial_answer/部分正确答案`:一个或多个选项,
    分隔符支持 `|`、`;`、`、`、`,`。
  - rules 文件格式:沿用现有三列 `item/response/score`(允许中文列名
    `题目/作答/分值` 自动映射)。
- 部分计分计分逻辑(内部统一转为 rules 后走现有 `irtc_score_rules`):
  - 有 `partial_answer` 的题:完全正确 → 2 分,部分正确 → 1 分,其余 → 0 分;
  - 满分声明(Q 矩阵 `max_score` ≥3)且需要更多档位时,用户须提供 rules
    明确各档分值,key 文件不做超过三档的隐式推断(避免歧义);
  - 无 `partial_answer` 的题:维持 0/1。
- 与 Q 矩阵一致性检查(在 `irtc()` 内、两者都提供时):
  - Q 声明分部计分但 key 无部分正确答案且无 rules → 警告(W423);
  - key 有部分正确答案但 Q 声明非分部计分 → 警告(W424);
  - 计分后观测最高分与 Q 声明满分不符 → 交 M4 极端类别流程处理并标注。

涉及文件:`R/irtc_score.R`、`R/irtc.R`、`man/irtc_score.Rd`、
新测试 `tests/testthat/test-score-partial.R`。

### M4 极端作答数据处理(新文件 R/irtc_rare_categories.R)

目标:稳健处理「Q 有但样本中无人作答的题」与「答案种类不完整的题
(无人正确/无人错误/无人部分正确)」。

- `irtc()` 新增参数 `rare_categories=c("collapse","prior")`。
- 整题无人作答(全 NA):沿用现有 W307 剔除机制,但结果侧升级:
  - `irtc_results()$items` 与报告的题目表中保留该题行,参数列为 NA,
    新增 `status` 列标记 `dropped_no_response`(未估计:无人作答),
    使 Q 矩阵声明的题目在产出中不"凭空消失"。
- 类别不完整(collapse,默认):
  - 现有 I124 连续化重编码已隐式折叠;V1.1 在其上显式化:
    当 Q 声明了满分/分部计分时,对比「声明类别集合」与「观测类别集合」,
    对每个未观测类别记录折叠映射(如 原始 0/1/2 → 观测 0/1,类别 1 未观测,
    2 并入 1),写入日志(新码 W425,逐题);
  - 题目参数输出新增 `categories_collapsed`(如 "1->0" / "无")与
    `max_score_declared` / `max_score_observed` 列;
  - 无人全对 → 最高类别不可估,折叠后按低一档满分估计并标注;
    无人全错 → 0 类不可估,重编码后基线上移并标注;
    无人部分正确 → 中间类别折叠,题目退化为 0/1 并标注
    (此时该题 GPCM 退化为 2PL 形式,属正常行为,报告中说明)。
- 类别不完整(prior,可选):
  - 保留声明的类别结构,对不可估阈值参数施加正态先验
    (默认 N(0, 2^2),可经 `control` 覆盖),经由核心层已有的
    xsi 先验机制(`irtc_mml_proc_prior_list_xsi`)传入,核心层零改动;
  - 输出中该阈值的 SE 标注为先验主导(`prior_dominated=TRUE` 列),
    报告中明确提示解释须谨慎。
- 两种方案的选择建议写入手册:常规分析用 collapse;
  跨年锚题链接、必须保持类别结构可比时用 prior。

涉及文件:新 `R/irtc_rare_categories.R`、`R/irtc.R`、`R/irtc_results.R`、
新测试 `tests/testthat/test-rare-categories.R`。

### M5 GPCM 难度输出语义标注 + Q 表头贯通(输出层)

目标:GPCM 题目难度输出标明哪一列是部分正确难度、哪一列是完全正确难度;
样本结果输出用 Q 矩阵维度名作能力值/标准误等表头。

- 题目参数表(`irtc_param_table`,供 `irtc_results`/`irtc_excel`/报告共用):
  - 现有 `AXsi_.Cat*` 保留(向后兼容);
  - 分部计分题新增语义列:三类别题 `b_partial`(部分正确难度,类别 1 阈值)
    与 `b_full`(完全正确难度,最高类别阈值);
    更多类别时命名 `b_step1..b_stepK`,并在表尾/图例注明
    `b_step1=首个部分正确难度 … b_stepK=完全正确难度`;
    Excel 与报告中的中文表头直接写「难度(部分正确)」「难度(完全正确)」;
  - 0/1 题不受影响。
- 样本(person)输出:
  - 提供 Q 矩阵时,多维模型的 `eap_dim1/se_dim1/...` 表头替换为
    Q 维度名:机器可读结果(`irtc_results`/`irtc_json`)使用
    `eap_<维度名>/se_<维度名>`(维度名做列名安全化处理),
    并在 `model_info` 增加 `dimension_names` 字段与原 dim 序号的映射;
    Excel 人员表与报告直接用 Q 表头(如 「能力值(代数)」「标准误(代数)」);
  - 未提供 Q 时行为与 1.0 完全一致。
- `irtc_results` schema 版本升至 "1.1":新增列均为「追加」,
  1.0 已有列名与语义不变;`docs/` 中的 schema 说明同步更新。

涉及文件:`R/irtc_excel.R`(`irtc_param_table`)、`R/irtc_results.R`、
`R/irtc_report.R`、`R/irtc_plot.R`(表头透传)、相应 man 与测试。

### M6 报告增强(irtc_report)

目标:report 更详细、更专业。新增两章,三种受众(decision/survey/stat)
均包含、详略分层。

- 新章「模型诊断」:
  - 收敛信息:迭代数、最终参数最大变化量、是否达到收敛准则、
    未收敛时的醒目警示与建议;
  - 信息准则表(deviance/AIC/BIC/N 参数)+ 通俗判读文字
    (何时可比、越小越好等);
  - EAP 信度及判读区间(如 ≥0.80 良好 / 0.70–0.80 可用 / <0.70 谨慎);
  - 逐题拟合详表(infit/outfit MNSQ 与 t),附判读标准
    (0.7–1.3 可接受等,与 `irtc_quality_thresholds()` 一致);
  - decision 版只保留一段结论式摘要,stat 版含全部数值表。
- 新章「数据处理透明度」:
  - 完整清洗日志表(双语,按步骤分组);
  - 题目对齐结果:Q 有样本无 / 样本有 Q 无的题目清单与处理方式;
  - 权重说明:是否加权、权重来源列、加权样本量、权重分布摘要
    (min/max/CV,CV 过大给出提示);
  - 类别折叠记录:逐题列出声明满分、观测类别、折叠映射;
  - 剔除题目清单及原因(无人作答/零方差),与题目表 `status` 列呼应;
  - 计分摘要:答案键/规则来源文件、部分计分题清单。
- 题目表、人员表表头按 M5 使用语义标注与 Q 维度名。

涉及文件:`R/irtc_report.R`、`R/plain_summary.R`(同步提及新信息)、
测试 `tests/testthat/test-report.R` 扩展。

## 三、不做什么(边界)

- 核心估计算法、`irtc.mml`/`irtc.mml.2pl` 签名与行为:不改。
- 报告不新增嵌入图表、不新增方法学/文献章节(本轮已确认不做)。
- 题内多维(within-item multidimensionality)的 Q 矩阵 UI:不做,
  仅支持 between-item 简单结构(载荷可为非 0/1 数值,行为与核心层一致)。
- 自动模型选型:仍不做。

## 四、实施顺序与依赖

1. M1 权重(独立,最小风险)
2. M2 Q 矩阵读取与对齐(M3/M4/M5 的地基)
3. M3 答案文件与部分计分
4. M4 极端类别处理(依赖 M2 的声明信息)
5. M5 输出语义标注与表头贯通(依赖 M2/M4 的元数据)
6. M6 报告增强(最后,消费前五项全部元数据)
7. 版本收尾:DESCRIPTION 1.1.0、NEWS.md、手册两份、llms.txt、示例

每个模块独立提交:实现 + man + 测试 + ASCII 转义检查为一个 commit 单元。

## 五、验收标准(与 1.0 同一水准)

- testthat 全绿,0 failure 0 error;预期警告一律 `expect_warning()` 断言;
- 覆盖率:整体 ≥90%;本轮新增/大改的关键文件
  (irtc_qmatrix.R、irtc_rare_categories.R、irtc_read.R、irtc_score.R、
  irtc_results.R、irtc_report.R)≥95%;
- `R CMD check --as-cran --no-manual` 0 ERROR 0 WARNING;
- 端到端冒烟新增场景:csv 作答(含权重列)+ xlsx Q 矩阵(含分部计分列)
  + csv 答案键(含部分正确答案)→ `irtc(model="GPCM")` →
  `irtc_results`/`irtc_excel`/`irtc_report`(zh/en)全链路;
  构造含「无人作答题 + 无人部分正确题 + 无人全对题」的极端数据用例,
  分别验证 collapse 与 prior 两条路径;
- 验证脚本:新建 `scripts/verify-release-1.1.R`(复制 1.0 脚本并更新
  关键文件清单与冒烟场景);
- R 源码 ASCII-only 复查;`Sys.setenv(LANGUAGE="en")` 约定保持。
