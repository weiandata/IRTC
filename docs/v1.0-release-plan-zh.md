# IRTC 1.0.0 发布计划（CRAN 就绪版）

日期：2026-07-16　负责人：Ma Kunxiang　目标版本：1.0.0

## 一、目标与原则

在 0.1.0 已验证的估计核心（`irtc.mml` / `irtc.mml.2pl`，624 项测试通过，
`R CMD check --as-cran` 0 ERROR 0 WARNING）之上，新增易用层，服务四类人群：

| 人群 | 入口 | 输出 |
|---|---|---|
| 普通调查工作人员 | `irtc("数据.xlsx", model=...)` 一行命令 | 通俗 Excel 表、Word/HTML 报告 |
| 专业统计人员 | 原有 `irtc.mml` 系列（完全不变） | 全参数表、拟合统计、anova |
| AI Agent | `irtc_check_data` → `irtc` → `irtc_results`/JSON | 稳定 schema 的 tidy 数据、结构化报错、llms.txt |
| 决策者 | 由前三类人群生成 | 报告中的结论摘要页（可用题量、质量分级、建议） |

原则：核心估计零新增依赖；所有新依赖进 Suggests，缺失时给出一键安装提示；
原 API 完全向后兼容；新函数统一 `irtc_*` 蛇形命名；输出内容 `lang = "zh"/"en"` 双语。

## 二、新增功能模块

### M1 数据输入层（普通人群 + AI 的地基）

- `irtc_read(x, ...)`：统一读取入口。
  - Excel `.xlsx/.xls`（readxl）；`.csv/.tsv/.txt` 自动识别分隔符与编码
    （UTF-8/GBK，base R 实现）；SPSS/Stata/SAS `.sav/.dta/.sas7bdat`（haven）；
    data.frame/matrix/tibble 直接进入清洗。
  - 自动清洗：识别并抽离 ID 列为 `pid`；字符列转数值；常见缺失码
    （99、-9、空串、空格等，可配置）转 NA；剔除全空行列；异常值警告。
    全部动作记入清洗日志（附着在结果对象上，报告中可追溯）。
- `irtc_score(resp, key, rules = NULL)`：原始作答计分。
  - 客观题：A/B/C/D 原始选项 + 答案键 → 0/1。
  - 多级计分：规则表（题目 × 选项 → 分值）→ 部分计分数据。
- `irtc_check_data(x, key = NULL)`：估计前预检。
  - 人类可读打印 + 机器可读结构（问题列表：code / severity / item / fix）。
  - 检查项：全对全错题、极端缺失、非法类别、重复 pid、样本量不足等。

### M2 新增统计量（题目质量表必需，零依赖）

- `irtc_ctt(resp)`：答对率/均值、点二列相关（题总相关）、Cronbach's alpha。
- `irtc_itemfit(mod)`：基于残差的 infit / outfit MNSQ 及 t 统计量。
- 质量评级引擎：综合答对率、区分度、拟合 → 四级评级
  （好 / 可用 / 需检查 / 建议修改），每题附通俗中文/英文原因说明。

### M3 一站式主函数

- `irtc(data, model, key = NULL, lang = "zh", ...)`
  - `data`：文件路径或 R 对象（内部走 M1）；`model` 必填
    （"1PL"/"2PL"/"PCM"/"GPCM"/"RSM"），不做自动选型以免误导；
    `...` 全部透传给 `irtc.mml`/`irtc.mml.2pl`，专业用户可完全覆盖。
  - 返回原 `irtc` 类对象（增附清洗日志、CTT、拟合），现有 S3 方法全兼容。
- `plain_summary(mod, lang)`：面向普通人员的分层文字摘要（先结论后细节）。

### M4 Excel 导出（特殊需求）

`irtc_excel(mod, dir = ".", lang = "zh", prefix = NULL, overwrite = FALSE)`
一次生成三个独立 .xlsx 文件（openxlsx，支持配色与格式）：

1. **题目质量表** `*_item_quality.xlsx`：题号、答对率（通俗难度标签）、
   区分度、拟合、质量评级（红黄绿配色）、通俗原因与处理建议。
2. **题目难度区分度表** `*_item_parameters.xlsx`：题号、模型、难度 b 及 SE、
   区分度 a 及 SE、各类别阈值、作答人数、施测轮次/年份字段、量尺约束说明。
   列结构固定并写入文档，保证跨年度等值链接时可直接对齐锚题。
3. **样本能力值表** `*_person_ability.xlsx`：pid、原始分、作答题数、
   各维度 EAP 及 SE、百分位、T 分数。纯平表，可直接粘贴进总样本表。

### M5 报告生成

`irtc_report(mod, file, format = c("docx", "html"), audience = c("survey", "stat", "decision"), lang = "zh")`

- docx 用 officer（Suggests）；html 为自包含单文件，图以 base64 内嵌，
  不依赖 pandoc。
- 图表（base R 零依赖）：Wright Map（题目难度-人群能力对照）、能力分布
  直方图、题目质量分级汇总图、每题 ICC 曲线。
- 三种受众版式：
  - decision：1–2 页结论摘要（测验整体质量、可用/需修改题量、决策建议）。
  - survey：通俗全文（每个统计量配一句白话解释）。
  - stat：完整技术报告（参数、SE、拟合、收敛信息、模型比较）。

### M6 AI Agent 支持

- `irtc_results(mod)`：稳定 schema 的 tidy 数据框列表
  （items / persons / fit / model_info / cleaning_log），schema 写入文档。
- `irtc_json(mod, file)`：JSON 导出（jsonlite，Suggests）。
- 结构化条件系统：新代码的所有错误/警告带类
  （如 `irtc_error_data_format`）与字段 code / reason / fix，Agent 可解析重试。
- `inst/llms.txt`：紧凑 API 说明 + 典型工作流（读→检→估→出表→出报告），
  供 Agent 一次读入。

### M7 鲁棒性加固

- 全部新函数入口做参数校验，报错信息双语、含修复建议。
- 边界用例：单题、全对/全错人与题、>90% 缺失、单人、超大类别数、
  混合编码文件、含公式/合并单元格的 Excel。
- 估计不收敛时降级为可操作警告（建议增大 maxiter、检查数据）而非晦涩报错。

## 四、实施顺序

1. M1 数据输入层（含测试与 fixtures）
2. M2 统计量与质量评级
3. M3 `irtc()` 主函数 + `plain_summary()`
4. M4 三个 Excel 导出
5. M5 Word/HTML 报告
6. M6 AI Agent 支持（结果 schema、结构化报错贯穿改造、llms.txt）
7. M7 鲁棒性加固 + 文档/测试补全 + `R CMD check --as-cran` 验收

依赖关系：M4/M5 依赖 M2/M3；M6 的结构化报错在 M1 起即按规范实现。

## 五、风险与对策

- officer/openxlsx 属较重 Suggests → 仅在调用对应功能时加载，
  缺失时提示 `install.packages(...)`，核心功能不受影响。
- 双语字符串 → 集中在内部字典文件（`R/irtc_i18n.R`），避免散落硬编码；
  源码文件保持 ASCII（中文用 \uXXXX 转义），满足 CRAN 可移植性检查。
- 跨年链接表 schema 一旦发布即冻结 → 1.0 前在手册中明确列定义并加
  schema 版本号字段。
